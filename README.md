# NoteSpark

NoteSpark is an Obsidian plugin that generates AI-powered quotes, reflections, and prompts directly inside your notes. Use simple triggers like `@notespark` or `@notespark[1]` in normal notes or daily note templates, and NoteSpark replaces them with clean markdown generated by Mistral AI.

## Usage

Add a standalone trigger anywhere in a markdown note:

```md
@notespark
```

or choose a prompt preset:

```md
@notespark[1]
@notespark[3]
```

NoteSpark replaces the trigger with markdown:

```md
> [!quote]
> Discipline is the bridge between intention and progress.
```

## Commands

- **Generate triggers in active note** replaces all standalone `@notespark` triggers in the active editor.
- **Regenerate quote** replaces the current selection, or the callout under the cursor, after asking which prompt preset to use.

## Settings

- Mistral API key
- Model, temperature, max output tokens, and request timeout
- Default prompt and default maximum length
- Output mode: Obsidian callout or plain text
- Default callout type
- Editable prompt presets with numeric IDs, prompt text, maximum length, and optional callout type
- Auto-generation for active markdown files
- Template folder to ignore

## Daily Notes

Add a trigger to a daily note template:

```md
# Daily Note

@notespark[1]
```

To keep the template file unchanged, set **Template folder to ignore** to your vault-relative template folder, for example `Templates`.

When a daily note is created from that template outside the ignored folder, NoteSpark detects the copied standalone trigger and replaces it in the daily note.

## Privacy and Network Use

NoteSpark sends only the configured prompt text to Mistral's chat completions API when generation is triggered. The Mistral API key is stored locally in this vault's plugin data. The plugin does not add telemetry and does not leave hidden metadata in notes.

## Development

Install dependencies:

```sh
npm install
```

Run tests:

```sh
npm test
```

Build the plugin:

```sh
npm run build
```

The Obsidian plugin ID is `notespark`. For local testing, use a plugin folder named `.obsidian/plugins/notespark` so the folder name matches `manifest.json`.
